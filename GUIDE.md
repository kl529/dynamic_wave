## 🎯 매일 하는 일 (1분 루틴)

**장 마감 후 (미국 시간 오후 4시 = 한국 오전 5~6시) 메인 페이지를 열면 됩니다.**

### 화면이 보여주는 것

| 화면 표시 | 해야 할 일 |
|-----------|-----------|
| 🚀 **매수 신호 발생! N주** | 해당 분할 현금으로 N주 시장가 매수 |
| 💰 **매도 신호 발생! N주** | 해당 분할 전량 시장가 매도 |
| ⏳ **관망 (대기 중)** | 아무것도 안 해도 됨 |
| 🛡️ **현금 보유 중** | 아무것도 안 해도 됨 (RSI 하락장 감지) |

### 매일 체크리스트

```
□ 메인 페이지 접속
□ "오늘 실행할 매매" 확인
□ 신호가 있으면 해당 주식 앱에서 실행
□ 실행했으면 "매매 기록" 탭에 기록
```

> 💡 **핵심**: 시스템이 종가 기준으로 모든 계산을 완료해 알려줍니다.
> 매일 1번 들어가서 신호 확인 후 그대로 실행하면 됩니다.

## 📊 전략 설명

### 현재 파라미터 (v3.1 — 2026-04-04 최적화)

전체 기간(2010~2026) 백테스팅으로 최적화된 값입니다.

| 모드 | 매수 조건 | 매도 조건 | 최대 보유 |
|------|-----------|-----------|-----------|
| **안전** (safe) | -3% 이상 하락 | +2% 수익 | 20 거래일 |
| **공세** (aggressive) | -5% 이상 하락 | +8% 수익 | 7 거래일 |
| **강세** (bull) | -3% 이상 하락 | +12% 수익 | 45 거래일 |
| **현금** (cash) | 매수 없음 | 즉시 전량 청산 | — |

**기본 설정**: 10분할, 재분할 주기 10거래일, RSI 자동모드

### RSI 자동 모드 전환 (Weekly RSI 기반)

매주 금요일 종가 기준 **14주 RSI**를 계산해 모드를 자동 결정합니다.

```
지난주 RSI < 이전주 RSI (하락 중):
  → RSI < 40이면: cash (하락장 — 매수 중단, 기존 포지션 청산)
  → 그 외: safe (방어 모드)

지난주 RSI > 이전주 RSI (상승 중):
  → RSI 55~65 구간: bull (강세 — 목표 +12%)
  → 그 외 상승: aggressive (공세 — 목표 +8%)
```

**모드 전환 시점**: 해당 금요일 데이터가 반영된 다음 거래일부터 적용

### 하이브리드 전략 (B&H + 동파법)

백테스팅 연구(2010~2026)에서 동파법 단독 누적 수익률이 -87%인 반면,
하이브리드는 +1.49억% (연평균 +173%)를 기록했습니다.

**핵심 아이디어**: RSI가 강세일 때는 그냥 들고 있는 게 낫다 (B&H)

```
B&H 진입 조건 (3가지 모두 충족):
  1. RSI 모드 = aggressive 또는 bull
  2. 현재 가격 > SMA50 (50일 이동평균)
  3. 마지막 전략 전환 후 10 거래일 이상 경과

B&H 탈출 조건 (즉시 — 비대칭):
  - RSI 모드 = safe 또는 cash → 즉시 전량 매도 후 동파법 재진입
```

> **하이브리드 활성화**: 백테스팅 페이지 → RSI 자동모드 선택 → "하이브리드 전략" 토글 ON

### 매매 방식 — LOC (Limit-on-Close)

**종가 기준 지정가 주문**입니다.

```
매수: 당일 종가 < 전일 종가 × (1 + buyTarget)이면
      → 당일 종가로 매수

매도: 당일 종가 >= 평단가 × (1 + sellTarget)이면
      → sellTarget 적용 지정가로 매도 (종가가 더 높아도 지정가)

손절: 보유 거래일 >= holdingDays이면
      → 당일 종가로 강제 매도 (시장가)
```

**예시** (안전모드 buyTarget=-3%, sellTarget=+2%):
- 전일 종가 $100 → 오늘 $96.5 (-3.5%) → **매수 발동** ($96.5에 체결)
- 평단가 $96.5 → 매도지정가 $98.43 → 오늘 $99 → **$98.43에 LOC 매도**

### 수수료

```
수수료율 = 증권사 0.044% + SEC 0.00278% = 편도 0.047%
왕복 = 약 0.094%
```

안전모드 순수익: +2% - 0.094% = **+1.906% / 거래**

### 분할별 독립 운영

```
✅ 올바른 이해:
  분할1: { 매수일: 3/1, 보유: 23주, 평단가: $42.10, 보유 5일 }  ← 조건 충족 시 이것만 매도
  분할2: { 매수일: 3/3, 보유: 22주, 평단가: $43.50, 보유 3일 }
  분할3~10: { 현금: $1,000, 비어있음 }

❌ 잘못된 이해:
  전체 보유량 = 45주, 전체 평단가 = $42.8 → 한번에 전량 매도
```

### 퉁치기 (Netting)

같은 날 매수 + 매도가 동시 발생하면 **순매매량만 거래**해 수수료 절감:

```
분할3 매도: 22주
분할7 매수: 18주
→ 실제 실행: 22-18 = 4주만 매도 (수수료 82% 절약)
```

## 🏗️ 시스템 구조

### 핵심 파일

```
src/
├── app/
│   ├── page.tsx                  ← 메인 (오늘 매매 신호)
│   └── backtest/page.tsx         ← 백테스팅
│
├── services/
│   ├── divisionEngine.ts         ← ⭐ N분할 매매 엔진 + 하이브리드 B&H
│   └── dongpaEngine.ts           ← 레거시 단순 엔진
│
├── utils/
│   ├── tradingConfig.ts          ← ⭐ 모드별 파라미터 (v3.1)
│   └── rsiCalculator.ts          ← 주간 RSI 계산 + 모드 결정
│
├── constants/index.ts            ← TRADING 상수 (UI 표시용)
└── types/index.ts                ← 타입 정의
```

### DivisionEngine — 모드별 동작

```typescript
// 일반 동파법 백테스트
const engine = new DivisionEngine({ mode: 'auto', ... });
engine.backtest(data, modesByDate);

// 하이브리드 B&H + 동파법
const engine = new DivisionEngine({ mode: 'auto', hybrid: true, ... });
engine.backtest(data, modesByDate);
// → RSI aggr/bull + SMA50 조건 충족 시 자동 B&H 전환
// → RSI safe/cash 시 즉시 동파법 복귀
```

### 데이터 흐름

```
Yahoo Finance API → historicalData (일별 OHLCV)
  → enrichDataWithWeeklyRSIMode() → 날짜별 RSI 모드 맵
  → DivisionEngine.backtest() → DailyTradeRecord[]
  → UI 렌더링 (차트, 통계, 신호)
```

## 💻 개발 가이드

### 설치 & 실행

```bash
npm install
npm run dev    # http://localhost:3000
npm run build  # 프로덕션 빌드
```

### 파라미터 변경 방법

`src/utils/tradingConfig.ts` 의 `MODE_CONFIGS` 수정:

```typescript
const MODE_CONFIGS = {
  safe: {
    sellTarget: 0.02,     // +2% 수익 시 매도
    buyTarget: -0.03,     // -3% 이상 하락 시 매수
    holdingDays: 20,      // 최대 20 거래일
  },
  // ...
};
```

`src/constants/index.ts` 의 `TRADING` 도 동일하게 업데이트해야 UI가 맞습니다.

### 백테스팅 활용

1. 백테스팅 페이지 접속
2. 기간 설정 (최소 1년 이상 권장)
3. 모드: `RSI 자동모드` 선택
4. 하이브리드 전략 토글 ON/OFF로 비교 가능
5. "고급 설정"에서 파라미터 커스텀 후 스윕 실행

### 매매 기록 관리

- Supabase 연동 시: 자동 클라우드 저장 (여러 기기 공유)
- 미연동 시: localStorage 로컬 저장

## ⚠️ 투자 유의사항

- SOXL은 **3배 레버리지 ETF**로 일반 주식보다 변동성이 3~5배 큼
- 동파법은 **하락 매수 전략** — 장기 하락장에서 손실 누적 가능
- 하이브리드 전략은 백테스팅 결과이며 미래 수익을 보장하지 않음
- 본 시스템은 **교육 및 분석 목적**으로 제작됨
