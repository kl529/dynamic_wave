## 1. 매수 조건 (가장 큰 차이)

### 원본

```
LOC 매수목표가 = 전일 종가 × (1 + 매수점%)
                              ↑ 양수 (예: +3%)

체결 조건: 당일 종가 ≤ LOC 매수목표가

안전모드: 당일 종가 ≤ 전일종가 × 1.03  →  전일보다 최대 3% 상승해도 매수
공세모드: 당일 종가 ≤ 전일종가 × 1.05  →  전일보다 최대 5% 상승해도 매수
```

### 현재 (`divisionEngine.ts:233`)

```typescript
if (changeRate > modeConfig.buyTarget) return null;
// buyTarget = -0.04 (safe), -0.05 (aggressive)

// 즉: changeRate > -0.04 이면 매수 안 함
//     changeRate ≤ -0.04 이면 매수
// → 전일 대비 -4% 이상 하락해야만 매수
```

### 차이의 영향

| 시나리오 | 원본 | 현재 |
|---------|------|------|
| 전일 대비 +2% 상승 | 안전모드 매수 ✅ | 매수 없음 ❌ |
| 전일 대비 +0% 보합 | 안전모드 매수 ✅ | 매수 없음 ❌ |
| 전일 대비 -2% 하락 | 안전모드 매수 ✅ | 매수 없음 ❌ |
| 전일 대비 -4% 하락 | 안전모드 매수 ✅ | 안전모드 매수 ✅ |
| 전일 대비 -5% 하락 | 안전/공세 매수 ✅ | 안전/공세 매수 ✅ |

**결론**: 원본은 하락/보합/소폭상승일 모두에서 매수 → 회전율 매우 높음.
현재는 큰 하락일에만 매수 → 회전율 낮음. 완전히 다른 전략.

## 2. 매도 목표 (sellTarget)

### 원본 시트 기준

| 모드 | 매도 목표 | 이유 |
|------|----------|------|
| 안전 | **0.2%** | 높은 회전율 → 작은 수익 × 많은 횟수 |
| 공세 | **2.5%** | 추세 추종, 7일 내 청산 |

### 현재 `tradingConfig.ts` (v3 최적화 결과)

| 모드 | 매도 목표 | 비고 |
|------|----------|------|
| safe | **0.8%** | 원본의 4배 |
| aggressive | **6%** | 원본의 2.4배 |
| bull | **12%** | 원본에 없는 모드 |

**왜 올렸나**: 안전모드 0.2%는 수수료(0.094%) 대비 마진이 너무 작아서
백테스트에서 수수료를 감당 못 함. 하지만 현재 -4% 하락 진입 조건과 조합하면
0.8%도 달성이 어려울 수 있음 — 두 파라미터가 함께 최적화된 결과.

## 3. 모드 시스템

### 원본: 2모드

| 모드 | 전환 조건 |
|------|----------|
| safe | RSI > 65에서 **하락** / RSI 50 하향 돌파 / 40~50에서 하락 |
| aggressive | RSI < 35에서 **상승** / RSI 50 상향 돌파 / 50~60에서 상승 |

### 현재: 4모드 (`rsiCalculator.ts:determineWeeklyMode`)

| 모드 | 전환 조건 | 원본 대비 |
|------|----------|----------|
| cash | RSI < 40 + 하락 중 → 전량 청산 | 원본에 없음 (**추가**) |
| safe | 그 외 하락 중 / RSI > 65 | 조건 단순화 |
| bull | RSI 55~65 + 상승 중 | 원본에 없음 (**추가**) |
| aggressive | 그 외 상승 중 | 조건 넓어짐 |

### 세부 조건 차이

**안전모드 전환 조건:**

| 조건 | 원본 | 현재 |
|------|------|------|
| RSI > 65 | **하락 중일 때** safe | **방향 무관** safe (65 넘으면 무조건) |
| 40~50 하락 | 별도 명시 | 통합됨 (하락이면 다 safe) |

**공세모드 전환 조건:**

| 조건 | 원본 | 현재 |
|------|------|------|
| 상승 중 RSI 범위 | 50~60 | 30~60 (더 넓음) |

## 4. LOC 매도 체결가

### 원본

```
매도는 LOC 지정가 주문: 당일 종가 ≥ 목표가이면 → 목표가에 체결
(종가가 더 높아도 목표가에서 멈춤)
```

### 현재 (`divisionEngine.ts`)

```typescript
// 조건 2: 수익 목표 달성 → 종가에 매도
if (todayClose >= sellLimitPrice) {
  price: todayClose  // 목표가가 아닌 종가에 체결
```

**영향**: 종가가 목표가보다 높을수록 현재 구현이 더 유리함.
원본의 지정가 방식은 목표가 이상 상승분을 포기하지만, 브로커 실제 동작과 일치.
현재는 종가 = 당일 최고 근사치로 가정하는 단순화.

## 5. 목표 수량 계산

### 원본

```
목표 수량 = floor(매수예정금액 / LOC 매수목표가)
                              ↑ 한도가(높은 가격) 기준으로 수량 결정
실제 체결은 당일 종가(≤ 한도가)에 됨 → 약간의 현금 남음
```

### 현재 (`divisionEngine.ts:238`)

```typescript
const quantity = Math.floor(division.cash / (todayClose * (1 + getTotalFeeRate())));
//                                               ↑ 실제 체결가 기준 + 수수료 포함
```

**영향**: 원본은 한도가 기준이라 실제 체결 후 약간의 현금이 남음.
현재는 체결가 기준이라 현금 최대한 사용. 미미한 차이.

## 6. 엔진 구조 (단일 vs 다중 포지션)

### 원본: 7분할 독립 운영

```
분할1: HOLDING $30.00 → 목표가 $30.06 대기 중
분할2: HOLDING $28.50 → 목표가 $28.56 대기 중
분할3: HOLDING $27.00 → 목표가 $27.05 대기 중
분할4: EMPTY → 오늘 매수 가능
...
→ 같은 날 매수+매도 동시 발생 → 퉁치기 적용
```

### 현재: 두 엔진이 공존

| 엔진 | 위치 | 다중 포지션 | 퉁치기 | 사용처 |
|------|------|-----------|--------|--------|
| `DivisionEngine` | `divisionEngine.ts` | ✅ 원본과 동일 | ✅ 구현됨 | 백테스트 |
| `DongpaEngine` | `dongpaEngine.ts` | ❌ 단일 포지션 | ❌ 없음 | 실시간 신호 |

**DongpaEngine 단일 포지션 코드** (`dongpaEngine.ts:281`):

```typescript
if (매수신호.신호 === 'BUY' && 현재보유량 === 0) {
//                              ↑ 보유량이 있으면 매수 안 함
```

→ DongpaEngine은 원본 구조와 다름. 하지만 실제 백테스트/운영은 DivisionEngine 사용.

## 8. 복리 적용 방식

### 원본
```
10거래일 누적 손익 → 이익 × 80% 또는 손실 × 30% → 시드 조정
```

### 현재 (`dongpaEngine.ts:304`)
```typescript
// 매 거래 후 즉시 복리 누적
갱신복리금액 = 당일실현손익금액 * profitReinvest (or lossReinvest)
누적손익 += 갱신복리금액

// 10거래일마다 시드 갱신
if (자금갱신) 현재시드 = 초기자금 + 이전기록.누적손익;
```

DivisionEngine에서는 `rebalanceDivisions`가 `replenishPeriod`마다 호출됨.
전체적인 의도는 동일하나 누적 방식이 약간 다름.

## 핵심 결론

### 원본 전략의 설계 철학
```
회전율 극대화: 매일 매수 (보합/소폭상승 포함)
빠른 청산: 0.2% 도달하면 즉시 청산
7분할 동시 운영으로 항상 수익 기회 포착
퉁치기로 수수료 최소화
```

### 현재 구현의 변형 방향
```
선택적 진입: 큰 하락(-4%)일에만 매수 → 진입 품질 향상
높은 목표: 0.8~12% → 거래당 수익 증가
bull/cash 모드 추가 → 추세 대응 강화
분할 수 증가(7→10) → 더 세밀한 분산
```

### 어떤 게 더 나은가?
직접 비교 백테스트가 없어 단정할 수 없음.
원본(높은 회전율 + 낮은 목표)과 현재(낮은 회전율 + 높은 목표)는
서로 다른 시장 환경에서 유리할 수 있음.

**원본이 유리한 시장**: 횡보 / 완만한 상승 (매일 매수 가능)
**현재가 유리한 시장**: 변동성 큰 하락/반등 (큰 하락 후 반등 포착)
