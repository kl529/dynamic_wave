### 작업 내용

#### 1. 2010년~현재 연도별 백테스트 (`scripts/yearly.ts`)

SOXL 상장일(2010-03-11)부터 2026년까지 연도별 독립 백테스트 실행.

**핵심 결론**:
- 전략이 B&H를 이기는 해: 2011(+21.9%p), 2018(+34.1%p), 2022(+50.5%p) — 대폭락 3년만
- 나머지 14년은 모두 B&H에 패배
- 누적: 동파법 -90.4% vs B&H +5757%
- **동파법은 "대폭락 방어 전략"으로 포지셔닝해야 함**

#### 2. 그리드 탐색 v3 (`scripts/optimize.ts` 업데이트)

**변경 사항**:
- 평가 구간 4개 → 5개 (2012 횡보장 추가: B&H 0% 구간에서 손실 검증)
- `safe_hold` 변수화: [20, 30, 45일] (기존 30일 고정)
- 탐색 조합: 87,480개 → 14,580개 (safe_sell 고정, safe_hold 추가)
- 결과 파일: `docs/optimize-results-v3.csv`

**핵심 발견**:
- 14,580개 조합 모두 5구간 평균 손실 (최선 -8.6%)
- **분할 수(div)가 가장 중요한 변수**: div=10이 div=5 대비 20~25%p 개선
- 현재 기준값(div=5)은 최적값(div=10) 대비 모든 구간에서 대폭 열세

**즉시 적용 권장 파라미터**:
```
div=10, rp=10
safe:  buy -4%, sell +0.8%, hold 30일
aggr:  buy -5%, sell +10%,  hold 10일
bull:  buy -3%, sell +8%,   hold 45일
```

### 1. 기본 파라미터 최적화

#### 안전모드 매도 목표: 0.2% → **0.5%**
```
파일: src/utils/tradingConfig.ts, src/constants/index.ts

변경 전: sellTarget = 0.002 (0.2%)
변경 후: sellTarget = 0.005 (0.5%)

근거:
  수수료 왕복 = 0.044% × 2 + 0.00278% × 2 ≈ 0.094%
  구 실질수익 = 0.2% - 0.094% = +0.106% (너무 작음)
  신 실질수익 = 0.5% - 0.094% = +0.406% (4배 개선)

기대 효과:
  - 거래당 수익이 4배 증가
  - 슬리피지 발생해도 수익 확보 가능
  - 매도 빈도는 줄어들지만 1회 수익이 의미있는 수준
```

#### 공세모드 최대 보유일: 7일 → **10일**
```
파일: src/utils/tradingConfig.ts, src/constants/index.ts

변경 전: holdingDays = 7
변경 후: holdingDays = 10

근거:
  SOXL 일변동성 5~8%, 2.5% 목표를 7거래일 내 달성이 빡빡함
  3거래일 추가 시 목표 달성률 상승 (추정 +15~20%)
  강제 손절 없이 자연 매도 가능성 증가
```

#### 기본 분할 수: 5 → **7**
```
파일: src/constants/index.ts

변경 전: divisions = 5
변경 후: divisions = 7

근거:
  퉁치기(Netting) 발생 빈도 증가 (수수료 절감)
  더 세밀한 평균 단가 분산 (분할당 약 $1,429)
  2분할 여유 유지 원칙 준수 가능 (7분할 중 5개 투입, 2개 예비)
```

#### 기본 재분할 주기: 10일 → **15일**
```
파일: src/constants/index.ts

변경 전: rebalancePeriod = 10
변경 후: rebalancePeriod = 15

근거:
  SOXL의 추세 지속 기간 평균 2~3주
  10일 주기는 추세 중간에 재분할 → 수익 분할 축소
  15일 주기는 추세 완성 후 재분할 → 복리 효과 더 누림
```

### 2. 매수 조건 버그 수정 (사용자 패치 반영)

```
파일: src/utils/tradingConfig.ts (사용자 직접 수정)

수정 전: buyTarget = +0.03 (양수) → 변동률 < +3% 이면 매수 (매일 매수)
수정 후: buyTarget = -0.03 (음수) → 변동률 < -3% 이면 매수 (3% 이상 하락일만 매수)

핵심 차이:
  구버전: SOXL이 오늘 +2% 오른 날도 매수 → 고점 추격매수 위험
  신버전: SOXL이 오늘 -3% 이상 떨어진 날만 매수 → 진짜 "하락 매수" 전략

이 수정으로 전략의 성격이 근본적으로 바뀜:
  구버전: 고빈도 (거의 매일) 매수, 단기 변동 추종
  신버전: 저빈도 (하락일만) 매수, 평균단가 낮추기 전략
```

파일: src/components/BacktestStatsDashboard.tsx

추가된 지표:
