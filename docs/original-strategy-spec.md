## 4. 매수 로직 (핵심)

### 4-1. LOC 매수목표가 계산

```
LOC 매수목표가 = 전일 종가 × (1 + 매수점%)

안전모드: 전일 종가 × 1.03
공세모드: 전일 종가 × 1.05
```

### 4-2. 체결 조건

```
당일 종가 ≤ LOC 매수목표가 → 매수 체결 (종가에 체결)
당일 종가 > LOC 매수목표가 → 미체결
```

> **중요**: 이것은 "N% 이상 하락해야 매수"가 아님.
> 전일 대비 최대 N%까지 상승해도 매수됨.
> 즉, 가격이 크게 오른 날만 제외하고 대부분 매수됨.

### 4-3. 목표 수량 계산

```
목표 수량 = floor(매수예정금액 / LOC 매수목표가)
실제 체결 수량 = 목표 수량 (종가로 체결)
```

수량을 LOC 매수목표가(한도가) 기준으로 계산해두고,
실제로는 종가(≤ 한도가)에 체결되므로 약간의 현금이 남는다.

### 4-4. 실제 예시

```
안전모드, 전일 종가 $26.45

LOC 매수목표가 = $26.45 × 1.03 = $27.24
목표 수량 = floor($1,428.57 / $27.24) = 52주

케이스 A: 당일 종가 $27.08 (전일 대비 +2.4%)
  → $27.08 ≤ $27.24 → 체결 ✅ (상승일이지만 체결)

케이스 B: 당일 종가 $27.30 (전일 대비 +3.2%)
  → $27.30 > $27.24 → 미체결 ❌
```

## 5. 매도 로직

### 5-1. LOC 매도 (정상 매도)

```
LOC 매도목표가 = 매수가 × (1 + 매도점%)

안전모드: 매수가 × 1.002  (0.2% 수익)
공세모드: 매수가 × 1.025  (2.5% 수익)

체결 조건: 당일 종가 ≥ LOC 매도목표가
체결 가격: 매도목표가 (종가가 더 높아도 지정가에 체결)
```

### 5-2. MOC 강제 매도 (최대 보유기간 초과)

```
보유 거래일 ≥ 최대 보유기간 → 당일 종가로 강제 매도

안전모드: 30 거래일 초과 시
공세모드: 7 거래일 초과 시
```

### 5-3. 실제 예시

```
09.02 매수: 53주 @ $25.25, 목표가 = $25.25 × 1.002 = $25.30
09.03 매수: 54주 @ $24.91, 목표가 = $24.91 × 1.002 = $24.96

09.04 종가: $25.65
  → $25.65 ≥ $25.30 → 09.02 분할 매도 ✅ 수익 +$21.20
  → $25.65 ≥ $24.96 → 09.03 분할 매도 ✅ 수익 +$39.96
```

## 6. 퉁치기 (Netting) - 수수료 절감

같은 날 매수 신호와 매도 신호가 동시에 발생하면,
겹치는 수량만큼 수수료를 면제하는 브로커 기능.

```
예: 같은 날 52주 매수 + 53주 매도 발생 시
  겹치는 수량 = min(52, 53) = 52주
  52주에 대한 매수+매도 수수료 면제
  → 실질 수수료: 1주(매도)에만 부과
```

원본 시트에서는 "수수료절약" 컬럼으로 트래킹함.

## 9. 원본 전략의 특성

### 원본이 v3보다 더 자주 매수하는 이유

원본 안전모드에서 LOC 매수목표 = 전일종가 × 1.03.
즉, 전일보다 최대 3% 상승하지 않는 한 매수된다.
실제로 SOXL 일변동성 기준, 약 70~80%의 거래일에 매수 체결됨.

반면 v3는 -3% 이하 하락일에만 매수 → 약 15~20%의 거래일에만 매수됨.

### 원본이 낮은 매도 목표(0.2%)를 설정한 이유

매수 빈도가 높기 때문에 분할이 빠르게 채워짐.
그래서 빠르게 청산(0.2% 목표)하고 다음 사이클을 돌리는 것이 목적.
회전율이 핵심 수익 창출 메커니즘.

### 원본의 구조적 리스크

1. 매수가 자주 일어나므로 하락 추세에서 모든 분할이 빠르게 채워질 수 있음
2. 7분할이 모두 채워진 후 추가 하락 시 MOC 손절이 연속 발생
3. 안전모드 0.2% 목표는 수수료(약 0.094%) 대비 마진이 매우 작음
