## 1. 시스템의 전체 흐름

```
매일 장 마감 후 (미국 ET 4:00pm):
┌─────────────────────────────────────────────────────┐
│ 1. 오늘 SOXL 종가 확인 (Yahoo Finance API)            │
│ 2. 재분할 주기 체크 (N일째? → 자산 균등 재분배)          │
│ 3. 5개 분할 상태 업데이트 (평가금액, 지정가 계산)          │
│ 4. 각 분할별 매도 신호 체크 (수익 달성? 기간 초과?)        │
│ 5. 다음 순번 EMPTY 분할 매수 신호 체크                   │
│ 6. 퉁치기 계산 (같은 날 매수+매도면 수수료 절감)           │
│ 7. LOC 주문 실행 → 종가에 체결                          │
└─────────────────────────────────────────────────────┘
```

**핵심 원칙**: 하루에 딱 1분할만 매수 가능 (분할 포인터가 순번대로 회전)

## 2. 매수 메커니즘 완전 해부

### 2-1. 매수 신호 판단 로직 (코드 그대로)

```
매수 조건:
  (1) 해당 분할이 EMPTY 상태
  (2) 오늘이 그 분할의 "순번"
  (3) 분할 현금 ≥ $100 (MIN_CASH_FOR_TRADE)
  (4) 전일 대비 변동률 < buyTarget

변동률 = (오늘 종가 - 어제 종가) / 어제 종가
```

### 2-2. buyTarget의 실제 의미 (v3 기준)

```
안전모드: buyTarget = -0.03 (음수 -3%)
  → 오늘 변동률 < -3% 이면 매수
  → 즉, 오늘 -3%를 초과하여 하락해야만 매수

공세모드: buyTarget = -0.05 (음수 -5%)
  → 오늘 변동률 < -5% 이면 매수
  → 즉, 오늘 -5%를 초과하여 하락해야만 매수
```

**핵심**: 진짜 "하락 매수" 전략.
- 오늘 -4%: 안전모드 매수 ✅ (하락폭이 3% 초과)
- 오늘 -2%: 안전모드 매수 안 함 ❌ (하락폭 부족)
- 오늘 +2%: 안전모드 매수 안 함 ❌ (상승일)
- 오늘 -6%: 공세모드 매수 ✅ (하락폭이 5% 초과)

→ **"큰 하락일에만 진입"** — 매수 빈도는 낮지만 각 매수가 유리한 진입점

### 2-3. 매수 체결 수량 계산

```
수수료율 = 0.044% + 0.00278% = 0.04678% (편도)

매수 수량 = floor( 보유현금 / (오늘 종가 × (1 + 수수료율)) )

예시: 보유현금 $2,000 / SOXL $30.00
  수량 = floor(2000 / (30 × 1.0004678))
       = floor(2000 / 30.014034)
       = floor(66.63...)
       = 66주
```

### 2-4. 매수 후 분할 상태 변화

```
Before:
  분할1: EMPTY, 현금 $2,000

After (SOXL = $30.00, 66주 매수):
  분할1: HOLDING
  매수량:  66주
  체결가:  $30.00 (오늘 종가)
  매수금액: 66 × $30.00 = $1,980.00
  수수료:  $1,980.00 × 0.04678% = $0.93
  총 매수비용: $1,980.93 (= totalCost)
  잔여현금: $2,000 - $1,980 - $0.93 = $19.07
  평단가(avgPrice): $30.00
```

## 3. 매도 메커니즘 완전 해부

### 3-1. 매도에는 2가지 방식이 있다

#### 방식 A: 목표 수익 도달 → 지정가 LOC 매도 (SELL)

```
매도 지정가 = 평단가 × (1 + sellTarget)

안전모드 (sellTarget = 0.2%):
  예: 평단가 $30.00 → 매도 지정가 = $30.00 × 1.002 = $30.06

공세모드 (sellTarget = 2.5%):
  예: 평단가 $30.00 → 매도 지정가 = $30.00 × 1.025 = $30.75

체결 조건: 오늘 종가 ≥ 매도 지정가
체결 가격: 지정가 (오늘 종가가 더 높아도 지정가에 체결)
```

**LOC의 핵심**: 오늘 종가가 $30.50이어도, 지정가 $30.06에 체결.
→ 지정가 이상으로 오르면 항상 체결되지만, 추가 상승분은 포기.

#### 방식 B: 최대 보유기간 초과 → 시장가 손절 (STOP_LOSS)

```
보유 거래일 ≥ holdingDays 이면 오늘 종가로 강제 매도

안전모드: holdingDays = 30거래일 (약 6주)
공세모드: holdingDays = 7거래일 (약 1.5주)

보유 거래일 계산: 매수일 ~ 오늘까지 주말 제외 영업일 수
```

### 3-2. 매도 후 수익 계산

```
매도금액 = 매도 수량 × 체결가
매도 수수료 = 매도금액 × 0.04678%
순수익 = 매도금액 - 매도 수수료 - totalCost

수익률 = 순수익 / totalCost × 100
```

#### 예시: 안전모드 정상 매도

```
매수: 66주 @ $30.00, totalCost = $1,980.93
매도 지정가: $30.00 × 1.002 = $30.06

오늘 종가 $30.15 → 체결 (≥ $30.06)
매도 체결가: $30.06 (지정가)
매도금액: 66 × $30.06 = $1,983.96
매도 수수료: $1,983.96 × 0.04678% = $0.928

순수익 = $1,983.96 - $0.928 - $1,980.93 = +$2.10
수익률 = $2.10 / $1,980.93 × 100 = +0.106%
```

#### 예시: 안전모드 손절 (30거래일 경과)

```
매수: 66주 @ $30.00, totalCost = $1,980.93
30거래일 후 SOXL = $25.00 (하락)

강제 매도 체결가: $25.00 (오늘 종가)
매도금액: 66 × $25.00 = $1,650.00
매도 수수료: $1,650.00 × 0.04678% = $0.772

순수익 = $1,650.00 - $0.772 - $1,980.93 = -$331.70
수익률 = -$331.70 / $1,980.93 × 100 = -16.74%
```

## 4. 수익이 발생하는 정확한 수식

### 4-1. 1회 거래 순수익 공식

```
편도 수수료율 = f = 0.04678% = 0.0004678

매수: Q주 @ 체결가 P_buy
  매수금액: A_buy = Q × P_buy
  매수 수수료: C_buy = A_buy × f
  totalCost = A_buy + C_buy = A_buy × (1 + f)

매도: Q주 @ 매도 지정가 P_sell = P_avg × (1 + s)  [s = sellTarget]
  매도금액: A_sell = Q × P_sell
  매도 수수료: C_sell = A_sell × f

순수익 = A_sell - C_sell - totalCost
       = Q × P_sell × (1 - f) - Q × P_buy × (1 + f)
       = Q × [P_sell × (1 - f) - P_buy × (1 + f)]

P_avg = P_buy (단순 1회 매수의 경우 평단가 = 체결가)
P_sell = P_avg × (1 + s) = P_buy × (1 + s)

수익률 = 순수익 / totalCost
       = [P_buy × (1+s) × (1-f) - P_buy × (1+f)] / [P_buy × (1+f)]
       = [(1+s)(1-f) - (1+f)] / (1+f)
       = [(1+s-f-sf) - (1+f)] / (1+f)
       ≈ s - 2f  (sf ≪ 1 이므로 무시)
       = sellTarget - 2 × 수수료율
```

### 4-2. 손익분기 sellTarget

```
손익분기: s - 2f = 0
  s = 2f = 2 × 0.04678% = 0.09356% ≈ 0.094%

→ sellTarget이 0.094% 이상이면 수익
→ sellTarget이 0.094% 미만이면 손실 (수수료도 못 냄)
```

### 4-3. 실제 sellTarget별 기대수익 (분할당 $2,000 기준)

| sellTarget | 실질 수익률 | 거래당 이익 (66주 @ $30) |
|---|---|---|
| 0.2% | **+0.106%** | **+$2.10** |
| 0.5% | **+0.406%** | **+$8.04** |
| 1.0% | **+0.906%** | **+$17.93** |
| 1.5% | **+1.406%** | **+$27.82** |
| 2.5% | **+2.406%** | **+$47.60** (공세모드) |
| 3.0% | **+2.906%** | **+$57.49** |

## 5. 손실이 발생하는 조건

### 5-1. 손절 손실 (STOP_LOSS)

```
손절 손실률 = (P_stop / P_buy - 1) - 2f

P_stop = 손절 시 종가

예: 매수 $30.00, 30거래일 후 $25.00
  손절 손실률 = (25/30 - 1) - 0.094% = -16.67% - 0.094% ≈ -16.76%
```

SOXL 손절 패턴:
- SOXL이 30거래일 동안 회복 못 하면 → 강제 손절
- SOXL이 반도체 하락 사이클에 진입하면 5분할 모두 연속 손절 가능

### 5-2. 연속 손절 시나리오 (최악의 경우)

```
$10,000 자본, 5분할 (분할당 $2,000)

D+0:  분할1 매수 @ $40.00 (66주)
D+5:  분할2 매수 @ $35.00 (57주)   (-12.5% 하락)
D+10: 분할3 매수 @ $30.00 (66주)   (-14.3% 추가 하락)
D+15: 분할4 매수 @ $26.00 (76주)   (-13.3% 추가 하락)
D+20: 분할5 매수 @ $22.00 (90주)   (-15.4% 추가 하락)

30거래일 후:
  분할1 손절 @ $22.00 (5개 기간 겹칠 수 있음)
  손실: (22/40 - 1) - 0.094% = -45.09%
  → 분할1 손실: $2,000 × 45% = -$900

누적 손절 시: 전체 자본 $10,000 → $5,000~$7,000 수준까지 감소 가능
```

### 5-3. 돈을 잃는 3가지 근본 원인

| 원인 | 상황 | 대응 |
|---|---|---|
| 추세 하락 | SOXL이 지속 하락 → 30일 내 회복 불가 | RSI 공세모드 진입 금지 |
| 높은 진입가 | 고점에 5분할 모두 투입 | 항상 2분할 이상 여유 유지 |
| 낮은 sellTarget | 0.2% 목표 → 수수료 제외 실질 수익 거의 없음 | 최소 0.5% 이상 |

## 6. 퉁치기(Netting)로 수수료 절감

### 6-1. 퉁치기 발동 조건

```
같은 날: 분할A에서 매수 신호 + 분할B에서 매도 신호 동시 발생
```

### 6-2. 퉁치기 수수료 절감 계산

```
매수 70주 + 매도 40주 발생 시:
  겹치는 수량(nettedQty) = min(70, 40) = 40주

  퉁치기 적용:
    매수 수수료 factor = (70 - 40) / 70 = 42.86%  → 57.14% 절감
    매도 수수료 factor = (40 - 40) / 40 = 0%       → 100% 절감

  절감액:
    매수 측: 70주 × $30 × 0.04678% × 57.14% = $0.56
    매도 측: 40주 × $30 × 0.04678% × 100%   = $0.56
    총 절감: $1.12

  이 절감액은 매도 수익에 가산됨 (profit 증가)
```

### 6-3. 퉁치기의 실질 효과

```
하루에 매수+매도가 겹칠수록 수익률 향상.
분할 수가 많을수록 (7~10분할) 퉁치기 발생 빈도 증가.

5분할 기준 예상 퉁치기 발생 빈도: 월 3~5회
10분할 기준 예상 퉁치기 발생 빈도: 월 8~12회
```

## 7. 돈을 벌 수밖에 없는 구조 설계

### 핵심 원리: "작은 수익 × 많은 횟수 × 손실 제한 = 총 수익"

아래 조건을 모두 충족하면 장기적으로 수익이 날 수밖에 없다.

---

### 조건 1: sellTarget ≥ 손익분기점의 5배

```
손익분기: 0.094%
최소 sellTarget: 0.094% × 5 = 0.47% → 0.5%로 설정

이유:
  슬리피지 0.05% 발생해도 안전
  수익이 수수료의 5배 이상이어야 "확실한 플러스"
```

**즉시 변경 사항**: 안전모드 sellTarget을 0.2% → **0.5%**로 올려야 함

---

### 조건 2: 항상 2분할 이상 EMPTY 상태 유지

```
5분할 중 3개만 투입, 2개는 예비금으로 보유.
→ SOXL 하락 시 추가 매수 여력 확보
→ 급락 후 반등 시 저점 매수 가능

5분할 모두 HOLDING → 추가 기회 없음, 재분할까지 기다려야 함
```

**운영 규칙**: 한 번에 3분할 이상 동시 HOLDING 금지

---

### 조건 3: RSI에 따른 모드 전환 엄수

```
Weekly RSI 흐름에 따라:

  RSI 하락 중 (특히 > 65에서 하락): → 안전모드 강제
    sellTarget 0.5%, 매도 빠르게, 30일 내 청산

  RSI 상승 중 (특히 < 40에서 상승): → 공세모드 허용
    sellTarget 2.5%~3.0%, 추세 추종

결론:
  시장이 꺾일 때: 작은 수익으로 빠르게 회수
  시장이 오를 때: 수익 크게 가져가고 빨리 청산
```

---

### 조건 4: 재분할 주기를 시장 상황에 맞게

```
횡보/하락장: 재분할 주기 짧게 (5~10일)
  → 손실 분할을 빠르게 정리하고 균등 재배분

강한 상승장: 재분할 주기 길게 (15~20일)
  → 수익 분할의 복리 효과를 더 누림
```

---

### 조건 5: 손절 횟수 ÷ 목표수익 달성 횟수 < 0.3 유지

```
즉, 10번 거래 중 손절 3번 이하, 수익 7번 이상

이를 위한 매수 필터 강화:
  기본: 변동률 < +3% (거의 매일)
  강화: 변동률 < 0% (하락일에만 매수)
  최강화: 변동률 < -2% (2% 이상 하락일에만 매수)

강화할수록 매수 기회 감소 → 회전율 감소 → 하지만 승률 상승
```

### 7-1. 이론적 "최강 설정" (백테스트로 검증 필수)

```yaml
분할 수: 7분할
초기 자본: $10,000 (분할당 $1,429)
모드: RSI 자동 (Weekly RSI 기반)

안전모드 설정:
  매수 임계값: < +3%   (현재 유지)
  매도 목표:  0.5%    (0.2% → 0.5%로 상향)
  최대 보유:  30거래일 (현재 유지)

공세모드 설정:
  매수 임계값: < +5%   (현재 유지)
  매도 목표:  3.0%    (2.5% → 3.0%로 소폭 상향)
  최대 보유:  10거래일 (7일 → 10일로 소폭 연장)

재분할 주기: 15거래일

운영 규칙:
  - 동시 HOLDING ≤ 5분할 (7분할 중 2개는 항상 예비)
  - RSI 65 이상이면 공세모드 진입 금지
  - 손절 3연속 발생 시 1주일 관망 후 재개
```

### 7-2. 기대 수익 시뮬레이션 (이론값)

| 시장 상황 | 모드 | 월 예상 수익률 | 연 환산 |
|---|---|---|---|
| 강한 상승장 | 공세모드 (3.0%) | +8%~+20% | +96%~+240% |
| 완만 상승장 | 공세/자동 혼합 | +3%~+8% | +36%~+96% |

### 8-3. 손절 리스크 vs. 수익 기대값

```
기대값 = (승률 × 평균수익) - (패율 × 평균손실)

기대값이 양수 = 장기적으로 반드시 수익

예시 (안전모드, sellTarget 0.5%):
  승률 추정: 70% (30일 내 0.5% 달성)
  패율: 30% (30일 내 미달성 → 평균 -8% 손절)

  기대값 = (0.7 × 0.406%) - (0.3 × 8%) = 0.284% - 2.4% = -2.116%

→ 이 계산이면 손실!

안전모드 0.5% 설정이 수익이 되려면:
  승률이 최소 (8% × 0.3) / (8% × 0.3 + 0.406%) = 85% 이상 필요
  OR 손절 평균 손실 ≤ 1% 수준으로 낮춰야 함
```

**결론**: 단순히 sellTarget만 올린다고 해결되지 않는다.
**승률을 높이는 것** (= 하락 추세에 진입하지 않는 것)이 핵심.

## 최종 결론: 돈을 벌 수밖에 없는 1가지 핵심

```
수익 구조 = 승률 × 평균수익 > 패율 × 평균손실

이를 달성하기 위한 우선순위:

1순위: 손절 빈도 줄이기 (RSI 모드 전환 엄수, 공세모드 타이밍 철저)
2순위: sellTarget 높이기 (0.2% → 0.5%, 공세 2.5% → 3.0%)
3순위: 분할 여유 확보 (2~3분할 항상 EMPTY 유지)
4순위: 퉁치기 극대화 (분할 수 7~10으로 증가 검토)
5순위: 재분할 주기 최적화 (상승장 15일, 하락장 10일)
```

**가장 먼저 할 것**: 백테스팅에서 2022년~2023년 (하락+반등 전체) 기간으로
안전모드 sellTarget 0.2% vs. 0.5% vs. 1.0% 비교 실행.
수익률과 함께 **승률, 손절 비율** 확인이 핵심.
